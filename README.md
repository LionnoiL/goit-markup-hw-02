# WEBSTUDIO

## Description

The WEBSTUDIO project is the second homework assignment on HTML markup and CSS styling as part of
the GoIT Fullstack program. The project demonstrates the creation of a semantic
HTML structure for a digital solutions agency website with modular CSS styling architecture.

## Project Structure

```
goit-markup-hw-02/
├── index.html              # Main HTML file with markup
├── css/
│   ├── main.css            # Main CSS file (imports all other stylesheets)
│   ├── common.css          # Common styles (body, headings, links, buttons, etc.)
│   ├── header.css          # Header styling
│   ├── hero.css            # Hero section styling
│   ├── features.css        # Features section styling
│   ├── team.css            # Team section styling
│   ├── portfolio.css       # Portfolio section styling
│   └── footer.css          # Footer styling
├── images/                 # Directory with images
│   ├── team-1.jpg
│   ├── team-2.jpg
│   ├── team-3.jpg
│   ├── team-4.jpg
│   ├── portfolio-1.jpg
│   ├── portfolio-2.jpg
│   ├── portfolio-3.jpg
│   ├── portfolio-4.jpg
│   ├── portfolio-5.jpg
│   └── portfolio-6.jpg
└── README.md               # This file
```

## Website Content

### Header

- WEBSTUDIO company logo
- Navigation menu (Features, Team, Portfolio)
- Contact information (email and phone)

### Main

- **Hero section** — main heading with call-to-action button and dark background
- **Features section** — list of key company advantages with headings and descriptions
- **Team section** — team members presentation with photos, names, and job titles
- **Portfolio section** — showcase of company projects with images, titles, and categories

### Footer

- Company logo
- Business description
- Dark background styling

## Technologies

- **HTML5** — semantic markup
- **CSS3** — modular styling with separate files
- **Modern Normalize** — CSS reset library
- **Google Fonts** — Roboto and Raleway font families
- Proper document structure
- Use of semantic tags (`<header>`, `<main>`, `<footer>`, `<section>`, `<nav>`, `<address>`)

## Key Features Compared to HW-01

- **Modular CSS Architecture** — styles are organized into separate files by component
- **Enhanced Styling** — typography, colors, and layout are properly styled
- **Additional Content** — portfolio section with 6 projects
- **Color Scheme** — dark navigation, blue accents, and light backgrounds
- **Button Styling** — interactive button with hover effects
- **Link Styling** — styled navigation and contact links with hover effects

## Project Goal

This project is designed to practice the following skills:

- Creating semantically correct HTML markup
- Organizing content into logical sections
- Using anchor links for navigation
- Proper embedding of media elements
- Applying CSS styling to HTML elements
- Working with typography and colors
- **Organizing CSS with modular architecture**
- **Working with CSS imports and file structure**
- **Styling interactive elements (links, buttons) with hover states**
- Understanding CSS specificity and inheritance
- Basic responsive design considerations

## Running the Project

Open the `index.html` file in a web browser to view the website.
